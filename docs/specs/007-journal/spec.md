# Spec: 투자일기 (선택 작성 · 첫 매도 전 수정 잠금)

> 단계: **1차 고도화(이 문서의 2차 MVP)**. 현재 MVP 구현·통합 완료 조건에 포함하지 않는다.
>
> 선행: 현재 MVP 완료, `004-order-buy` (매수 체결), `005-order-sell` (매도 배분).
> plan.md·tasks.md는 착수 직전 작성한다.

## 개요

매수 체결 1건에 투자일기 1건을 선택적으로 작성한다. 목표가·손절가·근거 등 모든 항목이 선택이며, 해당 매수 lot에서 첫 매도 배분이 발생하기 전까지만 수정할 수 있다. 매매 전 복기 습관을 만드는 교육 기능이다.

## 사용자 시나리오

- 사용자는 매수 체결 후 원하면 그 체결에 투자일기를 남긴다 (안 남겨도 아무 제한 없음).
- 사용자는 아직 팔지 않은 매수 건의 일기를 자유롭게 수정한다.
- 사용자는 일부라도 매도한 매수 건의 일기를 수정하려 하면 거부당한다.
- 사용자는 본인의 일기 목록과 단건을 조회한다.

## 요구사항

### JOUR-001 선택 작성
- [ ] 매수 체결(buyTradeId) 1건당 일기 1건만 작성할 수 있다 (`UNIQUE(buy_trade_id)`, 중복 작성 409).
- [ ] 목표가·손절가·근거·예상보유기간·메모는 모두 선택 항목이다.
- [ ] 일기를 작성하지 않아도 매매·보유·커뮤니티 기능을 제한하지 않는다.
- [ ] 매도 체결에는 일기를 작성할 수 없다.

### JOUR-002 수정 잠금
- [ ] 해당 매수 lot에 매도 배분이 한 번도 없는 동안만 수정할 수 있다.
- [ ] 첫 매도 배분 이후에는 잔량이 남아 있어도 수정 불가 — 409 `JOURNAL_LOCKED`.
- [ ] 일기 삭제는 지원하지 않는다.

## 비즈니스 규칙

- 잠금 판정의 근거는 005의 배분(trade_allocations) 기록이다 — 별도 잠금 플래그를 저장하지 않고 배분 존재 여부로 판정한다 (plan에서 최종 확정).
- 본인의 매수 체결에만 작성·수정·조회할 수 있다 (타인 것 403).
- 일기 유무·내용은 주문·체결 로직에 어떤 영향도 주지 않는다.

## 범위 제외

- 일기 기반 AI 피드백·가격 알림 (투자일기 기본 기능 이후 별도 고도화).
- 일기 삭제 (지원 안 함).
- 매도 체결 일기·주간 회고 (1차 계약에 없음).

## 완료 조건

- [ ] 매수 체결에 일기 작성·수정·목록·단건 조회 통합 테스트 통과.
- [ ] 같은 매수 체결에 두 번 작성 시 409 테스트 통과.
- [ ] 첫 매도 배분 이후 수정 요청이 409 `JOURNAL_LOCKED`인 테스트 통과 (부분 매도 케이스 포함).
- [ ] 매도 전 수정은 허용되는 테스트 통과.
- [ ] 타인 일기 접근·수정 거부 테스트 통과.
- [ ] `./gradlew build` 통과.
