# Spec: 조회 (계좌 요약 · 보유자산 · 평가손익 · 주문 목록 · 거래내역)

> PRD 근거: ACCT-002, PORT-001, PORT-002, PORT-003. 선행: `004-order-buy`, `005-order-sell`.
> plan.md·tasks.md는 착수 직전 작성한다.

## 개요

계좌 요약(현금·평가액·손익), 보유자산 목록, 주문 목록, 거래내역을 조회 API로 제공한다. 평가값은 저장하지 않고 조회 시점의 최신 시세로 계산한다. QueryDSL 기반 목록 조회를 포함한다.

## 사용자 시나리오

- 사용자는 시장별 계좌의 현금잔고·보유평가액·총평가액·실현손익·미실현손익을 본다.
- 사용자는 보유 종목별 수량·평균단가·평가금액·미실현손익을 본다.
- 사용자는 자신이 요청한 주문의 매수·매도 구분, 종목, 요청수량, 주문유형, 상태와 요청시각을 본다.
- 사용자는 본인의 체결내역을 최신순으로 넘겨 보며, 매도 건에서 실현손익을 확인한다.

## 요구사항

### ACCT-002 계좌 조회
- [ ] 현금잔고, 보유평가액, 총평가액, 실현손익, 미실현손익을 시장별로 반환한다.
- [ ] 평가값은 저장하지 않고 최신 가격으로 계산한다.

### PORT-001 보유자산
- [ ] 계좌·종목별 보유수량과 평균단가를 제공한다.
- [ ] 평가금액과 미실현손익은 최신 시세로 계산한다.
- [ ] 전량 매도한 종목은 활성 보유목록에서 제외한다.

### PORT-002 거래내역
- [ ] 본인의 시장별 체결내역을 최신순으로 조회한다 (커서 페이지네이션).
- [ ] 체결가격·수량·거래금액·수수료·체결시각을 반환한다.
- [ ] 매도 내역은 FIFO 실현손익을 포함한다.

### PORT-003 주문 목록
- [ ] `GET /api/orders`로 본인의 주문만 최신순으로 조회한다.
- [ ] 주문 구분, 종목, 요청수량, 주문유형, 주문상태, 주문요청시각을 반환한다.
- [ ] 체결가격·체결금액·수수료·실현손익·체결시각은 중복 반환하지 않고 `GET /api/trades`에서 제공한다.
- [ ] 1차에서 정상 생성된 주문은 시장가 즉시 전량 체결된 `FILLED` 주문이다.
- [ ] MVP에서는 페이지네이션 없이 전체 주문을 반환한다. 페이지네이션은 실제 데이터 증가가 확인되면 후속 고도화로 분리한다.

## 비즈니스 규칙

- 조회는 원장(MySQL)과 최신 시세(003 계약)만 읽는다 — 평가값을 어디에도 저장하지 않는다.
- 타인의 계좌·보유·주문·거래내역은 조회할 수 없다 (본인 것만 — 위반 시 403 또는 404).
- 주문 목록은 “무엇을 요청했는가”, 거래내역은 “실제로 얼마에 체결됐는가”를 보여준다. 즉시 체결 구조에서도 두 원장의 의미를 섞지 않는다.
- 시세가 유효하지 않은 종목의 평가값 처리 방식은 plan에서 확정한다 (마지막 유효가 표기 등 — 체결과 달리 조회는 차단하지 않음).
