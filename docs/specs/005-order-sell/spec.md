# Spec: 시장가 매도 (FIFO lot 소비 · 실현손익)

> PRD 근거: ORD-001~004 (매도 관점), ORD-005, ORD-006. 선행: `004-order-buy` (매수 lot).
> plan.md·tasks.md는 착수 직전 작성한다.

## 개요

시장가 매도 주문을 받아 즉시 전량 체결하고, 실행시각이 가장 빠른 미소진 매수 lot부터 FIFO로 차감하며 배분(allocation)을 기록한다. 배분된 매수원가·수수료와 매도금액·수수료로 실현손익을 계산해 현금과 계좌 실현손익을 갱신한다.

## 사용자 시나리오

- 사용자는 보유한 주식·코인을 수량 지정 시장가로 매도하고, 체결 결과와 실현손익을 받는다.
- 사용자는 여러 번 나눠 산 종목을 한 번에 매도해도 가장 먼저 산 것부터 순서대로 원가가 계산되는 것을 확인한다.
- 보유수량보다 많이 팔려고 하면 거부된다.

## 요구사항

### ORD-001·002 시장가 전량 체결 (매도)
- [ ] 유효한 최신 가격 한 개로 즉시 전량 체결한다. 주문 가능 시간 규칙은 매수와 동일 (`MARKET_CLOSED`·`PRICE_UNAVAILABLE`).

### ORD-003 주문 검증 (매도)
- [ ] 수량 > 0, 시장별 수량 형식 규칙은 매수와 동일.
- [ ] 매도 수량 ≤ 보유수량. 초과 시 409 `INSUFFICIENT_QTY`.
- [ ] 검증 실패 시 주문·체결·잔고 변경을 일절 남기지 않는다.

### ORD-004 수수료와 현금 (매도)
- [ ] 매도 수수료율은 매수와 동일 (주식 0.015%, 코인 0.05%, 원 미만 내림).
- [ ] 매도 현금증가 = 거래금액 - 수수료.

### ORD-005 FIFO
- [ ] 매도는 실행시각이 가장 빠른 미소진 매수 lot부터 차감한다.
- [ ] 하나의 매도가 여러 매수 lot을 소비할 수 있으며 각 배분을 저장한다.
- [ ] 실현손익은 배분된 매수원가·매수수수료와 매도금액·매도수수료로 계산한다.
- [ ] 체결과 lot 배분 기록은 수정·삭제하지 않는다.

### ORD-006 멱등성
- [ ] 매수와 동일한 Idempotency-Key 규칙을 적용한다.

## 비즈니스 규칙

- 금액 계산에 double·float 금지 (PRD C-003).
- 주문+체결+FIFO 배분+holding 갱신+현금증가+실현손익 갱신은 하나의 트랜잭션이다 (PRD §7).
- lot의 잔여수량은 배분 합계와 항상 일치해야 한다 (최초수량 - 배분수량 합 = 잔여수량).
- 전량 매도된 종목은 holding에서 활성 목록 제외 처리한다 (조회 규칙은 006).
- 첫 매도 배분이 발생한 매수 lot은 투자일기 수정이 잠긴다 (잠금 판정 자체는 007 — 여기서는 배분 기록만 정확하면 됨).

## 범위 제외

- 매수 (004).
- 거래내역·실현손익 조회 API (006).
- 투자일기 잠금 로직 (007 — 배분 데이터만 제공).
- 세금 (1차 제외 — 수수료만).

## 완료 조건

- [ ] 하나의 매도가 여러 lot을 소비하는 케이스의 배분·실현손익 검증 테스트 통과.
