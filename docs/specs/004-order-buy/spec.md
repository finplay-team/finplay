# Spec: 시장가 매수 (검증 · 수수료 · 멱등성 · lot 생성)

> PRD 근거: ORD-001, ORD-002, ORD-003, ORD-004, ORD-006 (매수 관점). 매도는 `005-order-sell`.
> plan.md·tasks.md는 착수 직전 작성한다.

## 개요

시장가 매수 주문을 받아 유효한 최신 가격 하나로 즉시 전량 체결하고, 주문·체결·현금차감·보유(holding)·매수 lot 생성을 하나의 트랜잭션으로 처리한다. FIFO 손익 계산(005)의 원천이 되는 매수 lot이 여기서 만들어진다.

## 사용자 시나리오

- 사용자는 장중에 주식을 수량 지정 시장가로 매수하고, 즉시 체결 결과를 받는다.
- 사용자는 시세가 유효한 코인을 24시간 시장가로 매수한다.
- 사용자는 네트워크 재시도로 같은 요청을 두 번 보내도 한 번만 체결된다.
- 현금이 부족하거나 장이 닫혀 있으면 주문은 아무 흔적 없이 거부된다.

## 요구사항

### ORD-001 시장가 전량 체결 (매수)
- [ ] 유일한 주문유형은 `MARKET`이다. 지정가는 422 `UNSUPPORTED_ORDER_TYPE`.
- [ ] 요청은 시장, 종목ID, BUY, 수량, `Idempotency-Key` 헤더를 포함한다.
- [ ] 유효한 최신 가격 한 개로 즉시 전량 체결한다 — 부분 체결·슬리피지 없음.

### ORD-002 주문 가능 시간
- [ ] 주식은 Fixture 재생 장중에만 매수 가능. 장외는 409 `MARKET_CLOSED`, 예약 없음.
- [ ] 코인은 최신 시세가 유효하면 24시간 가능. 유효하지 않으면 409 `PRICE_UNAVAILABLE`.

### ORD-003 주문 검증 (매수)
- [ ] 수량 > 0. 주식 수량은 정수, 코인 수량은 소수점 8자리 이하.
- [ ] 코인 주문금액 최소 5,000원.
- [ ] 현금 ≥ 거래금액 + 수수료. 부족 시 409 `INSUFFICIENT_CASH`.
- [ ] 다른 시장 계좌·종목 조합 요청은 거부한다.
- [ ] 검증 실패 시 주문·체결·잔고 변경을 일절 남기지 않는다.

### ORD-004 수수료와 현금 (매수)
- [ ] 주식 매수 수수료는 거래금액의 0.015%, 코인은 0.05%.
- [ ] 수수료는 원 단위 미만 내림.
- [ ] 매수 현금차감 = 거래금액 + 수수료.

### ORD-006 멱등성
- [ ] 동일 사용자·동일 `Idempotency-Key` 재요청은 추가 체결 없이 최초 응답을 반환한다.
- [ ] 같은 키에 다른 본문이 오면 409 `IDEMPOTENCY_CONFLICT`.

## 비즈니스 규칙

- 금액·가격·수량 계산에 double·float 금지 — Java는 BigDecimal, DB는 DECIMAL 또는 원 단위 BIGINT (PRD C-003).
- 체결(trade) 기록은 수정·삭제하지 않는 불변 원장이다.
- 매수 체결 1건 = 매수 lot 1건 생성 (최초수량 = 잔여수량 = 체결수량).
- 주문+체결+현금차감+holding+lot 생성은 하나의 트랜잭션이다 (PRD §7).
- 체결가는 003 `PriceQueryService`의 유효한 최신 가격만 쓴다 — 다른 경로로 가격을 읽지 않는다.

## 범위 제외

- 매도·FIFO 배분·실현손익 (005).
- 지정가·예약·부분 체결·슬리피지 (1차 명시적 제외).
- 분산락·다중 인스턴스 동시성 (2차 — 1차는 단일 인스턴스 DB 트랜잭션 정합성).
- 보유자산·거래내역 조회 API (006).

## 완료 조건

- [ ] 수수료 계산(0.015%·0.05%, 원 미만 내림)과 현금차감액 검증 테스트 통과.
- [ ] 현금 부족·장외·시세 무효·수량 형식 위반 각각의 오류 코드와 "흔적 없음" 검증 테스트 통과.
- [ ] 동일 Idempotency-Key 재요청이 체결을 추가하지 않는 테스트, 다른 본문 409 테스트 통과.
- [ ] `./gradlew build` 통과.
