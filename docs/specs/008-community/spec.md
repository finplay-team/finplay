# Spec: 커뮤니티 (게시물 CRUD · 평면 댓글 · 소유권)

> PRD 근거: COM-001, COM-002, COM-003. 선행: `002-auth-account` (인증).
> plan.md·tasks.md는 착수 직전 작성한다.

## 개요

로그인 사용자가 텍스트 게시물을 쓰고 댓글로 소통하는 최소 커뮤니티다. 게시물은 CRUD와 최신순 페이지네이션, 댓글은 평면형 작성·조회·본인 삭제만 제공한다.

## 사용자 시나리오

- 사용자는 제목과 본문으로 게시물을 작성하고, 본인 게시물을 수정·삭제한다.
- 사용자는 게시물 목록을 최신순으로 넘겨 보고 단건을 조회한다.
- 사용자는 게시물에 댓글을 달고, 본인 댓글을 삭제한다.
- 사용자가 남의 게시물·댓글을 고치려 하면 거부당한다.

## 요구사항

### COM-001 게시물
- [ ] 로그인 사용자는 텍스트 제목·본문으로 게시물을 작성한다.
- [ ] 목록·단건조회·작성·본인 수정·본인 삭제를 제공한다.
- [ ] 목록은 최신순 페이지네이션(page·size)을 제공한다.

### COM-002 댓글
- [ ] 로그인 사용자는 게시물에 평면형 댓글을 작성한다.
- [ ] 게시물 상세에서 댓글 목록을 오래된 순으로 조회한다.
- [ ] 본인 댓글만 삭제할 수 있다.

### COM-003 권한
- [ ] 게시물·댓글 조회는 인증 사용자에게 허용한다.
- [ ] 수정·삭제는 소유자만 가능하다. 타인 콘텐츠 변경은 403 `FORBIDDEN`.

## 비즈니스 규칙

- 게시물 삭제 시 달린 댓글 처리 방식은 plan에서 확정한다 (함께 삭제가 기본 후보).
- 제목·본문·댓글 본문은 공백만으로 이뤄질 수 없다 (400 VALIDATION_ERROR).
- 비로그인 접근은 전부 401 — 공개 조회 없음 (PRD COM-003 "조회는 인증 사용자에게 허용").

## 범위 제외

- 수익 인증 분류·거래 첨부·좋아요·신고 (1차 제외).
- 댓글 수정·대댓글 (1차 제외).
- 이미지·파일 업로드 (1차 제외).
- 관리자 기능·게시물 검색 (1차 계약에 없음).

## 완료 조건

- [ ] 게시물 작성→목록(최신순 페이지네이션)→단건→수정→삭제 통합 테스트 통과.
- [ ] 댓글 작성→오래된 순 조회→본인 삭제 통합 테스트 통과.
- [ ] 타인 게시물 수정·삭제, 타인 댓글 삭제 각각 403 테스트 통과.
- [ ] 비로그인 접근 401 테스트 통과.
- [ ] `./gradlew build` 통과.
