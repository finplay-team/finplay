# ADR-0004: DB 스키마는 Flyway 마이그레이션으로만 변경한다

- 상태: 승인됨
- 날짜: 2026-07-22

## 맥락

`ddl-auto=update`로 하이버네이트가 스키마를 자동 변경하면 팀원마다 DB 상태가 달라지고, 운영 반영 시 어떤 DDL이 실행될지 예측할 수 없다. 팀 프로젝트에서 스키마 이력은 코드처럼 버전 관리되어야 한다.

## 결정

- 스키마 변경은 `src/main/resources/db/migration/V{N}__{설명}.sql` 파일로만 한다.
- `spring.jpa.hibernate.ddl-auto=validate` — 하이버네이트는 엔티티와 스키마가 일치하는지 검증만 한다. 불일치 시 기동 실패가 정상 동작이다 (마이그레이션 누락을 조기에 발견).
- 이미 머지된 마이그레이션 파일은 절대 수정하지 않는다. 고칠 게 있으면 새 버전 파일을 추가한다.
- 로컬/테스트(Testcontainers)/운영 모두 같은 마이그레이션을 실행한다.

## 결과

- 엔티티를 바꾸면 마이그레이션 파일도 같은 PR에 포함해야 한다 (안 하면 validate가 잡아낸다).
- 스키마 이력이 git에 남아 롤백 지점과 변경 사유 추적이 가능하다.
