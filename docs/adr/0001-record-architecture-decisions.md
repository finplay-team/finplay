# ADR-0001: 아키텍처 결정을 ADR로 기록한다

- 상태: 승인됨
- 날짜: 2026-07-22

## 맥락

팀 프로젝트에서 "왜 이렇게 만들었는지"가 구두로만 전달되면 팀원(그리고 AI 세션)이 같은 논의를 반복하거나 기존 결정과 충돌하는 구현을 하게 된다.

## 결정

구조에 영향을 주는 결정은 `docs/adr/NNNN-제목.md` 형식으로 기록한다.

- 형식: 맥락 → 결정 → 근거 → 결과(트레이드오프 포함).
- ADR은 한번 승인되면 수정하지 않는다. 뒤집을 때는 새 ADR을 만들고 기존 것을 "대체됨(superseded by ADR-NNNN)"으로 표시한다.
- AI(Claude)는 구현 전에 관련 ADR을 읽어야 하며, ADR과 충돌하는 구현 대신 새 ADR 초안을 제안해야 한다.

## 결과

- 결정의 근거가 남아 온보딩과 AI 세션 인수인계가 빨라진다.
- ADR 작성 비용이 들지만, 구조적 결정에만 적용해 부담을 줄인다.
