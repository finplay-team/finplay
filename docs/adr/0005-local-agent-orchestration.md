# ADR-0005: PR 리뷰/QA는 로컬 에이전트 오케스트레이션으로 한다

- 상태: 일부 대체됨 — 서브에이전트 로스터는 [ADR-0008](0008-four-agent-roster.md)이 대체 (6개 → 4개, planner 신설). 오케스트레이션 방식·스킬 진입점 결정은 유효.
- 날짜: 2026-07-22

## 맥락

PR이 올라오면 AI가 리뷰·QA를 수행하는 자동화가 필요하다. 공식 방식인 GitHub Actions + claude-code-action은 `ANTHROPIC_API_KEY` 시크릿이 필요하고 실행마다 API 토큰 비용이 발생한다. 팀은 API 키 없이 각자의 Claude Code 구독 세션으로 해결하기로 했다.

## 결정

- PR 리뷰/QA는 **로컬 Claude Code 세션이 오케스트레이터**가 되어 독립 서브에이전트(code-reviewer, test-runner, qa-verifier)에게 위임하는 방식으로 한다.
- 진입점은 스킬 두 개다.
  - `/feature <specs/NNN>` — 기능 개발 루프 (구현 → 테스트 → 빌드 → 리뷰 → QA)
  - `/review-pr <PR번호>` — PR 리뷰 오케스트레이션, 결과를 `gh pr review`로 게시
- 리뷰어로 지정된 팀원이 로컬에서 `/review-pr N`을 실행하는 것을 기본 흐름으로 한다.

## 한계 (알고 수용함)

- API 키 없이는 "PR 생성 즉시 무인 자동 리뷰"가 불가능하다. 서버에서 실행될 주체가 없기 때문이다.
- 준자동이 필요하면 세션을 켜둔 상태에서 `/loop 30m`으로 열린 PR을 폴링하는 방식을 쓸 수 있다 (선택 사항).
- 이후 팀이 API 키 예산을 확보하면 claude-code-action 기반 완전 자동화로 전환하는 새 ADR을 작성한다.

## 결과

- 비용 없이 팀원 전원이 동일한 리뷰 기준(에이전트 정의가 레포에 커밋됨)을 사용한다.
- 리뷰 실행이 사람의 명령 한 줄에 의존하므로, PR 템플릿 체크리스트와 브랜치 보호 규칙(리뷰 승인 필수)으로 실행을 강제한다.
